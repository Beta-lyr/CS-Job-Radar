# PRD：计算机学生职业方向雷达

## 1. 文档信息

**产品名称**：计算机学生职业方向雷达
**英文名**：CS Job Radar
**版本**：V1.0 MVP
**目标用户**：在校计算机相关专业学生、准备实习/校招的应届生、转码学生
**产品定位**：基于公开招聘数据与岗位要求分析，帮助计算机学生判断技术方向、理解市场需求、规划学习路线和简历项目。

---

## 2. 产品背景

计算机专业学生在实习、校招、转码过程中普遍面临一个问题：信息很多，但判断困难。

学生可以在 BOSS 直聘、牛客、OfferShow、职友集、看准网等平台看到岗位、薪资、面经、公司评价，但这些平台主要解决的是“找岗位”“查薪资”“看经验”。学生真正困惑的是：

1. 当前哪些技术方向机会更多？
2. Java、Go、前端、Android、AI 应用、测试开发哪个方向更适合自己？
3. 每个方向企业到底要求什么技能？
4. 应届生岗位和社招岗位要求有什么区别？
5. 自己应该学什么、做什么项目、简历怎么写？
6. 岗位变化趋势是否能反映未来学习方向？

因此，本产品不做传统招聘搜索，而是做“岗位数据分析 + 职业方向建议”。

---

## 3. 产品目标

### 3.1 核心目标

为计算机学生提供一个清晰、数据化、可执行的职业方向参考工具。

用户访问网站后，应能快速得到以下答案：

* 当前热门技术方向有哪些？
* 每个方向岗位数量、薪资、城市分布如何？
* 每个方向最常要求哪些技能？
* 哪些方向对应届生更友好？
* 选择某个方向后，应该学习哪些技术？
* 应该做什么项目来匹配招聘要求？

### 3.2 MVP 阶段目标

V1.0 不追求全量岗位搜索，不做完整招聘平台，只实现：

1. 聚合公开岗位数据样本。
2. 对岗位进行方向分类和技能抽取。
3. 生成方向趋势、薪资区间、技能要求、应届友好度等分析。
4. 面向学生输出学习建议、项目建议、简历建议。
5. 每周生成一份计算机技术岗位趋势报告。

---

## 4. 产品边界

### 4.1 本阶段做什么

V1.0 重点做：

* 技术方向趋势分析；
* 岗位技能词频分析；
* 城市与薪资分析；
* 应届友好度分析；
* 方向详情页；
* 每周岗位趋势报告；
* 学生学习建议；
* 简历项目建议。

### 4.2 本阶段不做什么

V1.0 暂不做：

* 用户投递岗位；
* 企业发布岗位；
* HR 沟通；
* 简历上传与自动优化；
* 会员付费系统；
* 大规模岗位全文搜索；
* 实时爬取全网招聘数据；
* 绕过登录、验证码、风控的数据采集；
* 直接复制商业招聘平台的完整岗位详情页。

## 5. 目标用户

### 5.1 核心用户

#### 用户 A：大二/大三计算机学生

特点：

* 已经学过基础课程；
* 不知道该选前端、后端、客户端、AI、测试还是数据方向；
* 想提前准备实习；
* 需要明确学习路线和项目方向。

核心诉求：

* 哪个方向更有前景？
* 我现在该学什么？
* 什么项目能写进简历？

#### 用户 B：大四/研二应届生

特点：

* 正在准备秋招或春招；
* 已有一些项目，但不知道是否匹配岗位；
* 对薪资、城市、方向选择敏感；
* 需要快速补齐技能短板。

核心诉求：

* 哪些方向应届岗位更多？
* 我的项目能不能匹配岗位？
* 简历应该突出哪些技能？

#### 用户 C：转码/跨专业学生

特点：

* 时间有限；
* 信息来源混乱；
* 容易被培训路线误导；
* 更需要基于真实岗位的路线判断。

核心诉求：

* 现在学哪个方向性价比最高？
* 最小可行学习路径是什么？
* 哪些技能是必须学，哪些只是加分项？

## 6. 核心用户痛点

| 痛点          | 说明                          | 产品解决方式            |
| ----------- | --------------------------- | ----------------- |
| 不知道选什么方向    | 学生无法判断 Java、前端、AI、测试等方向机会大小 | 提供方向热度、薪资、应届友好度对比 |
| 不知道岗位真正要求什么 | JD 内容长且混乱，必备项和加分项难区分        | 抽取高频技能、职责、项目能力    |
| 不知道该做什么项目   | 学生项目同质化严重，简历缺乏亮点            | 根据岗位需求推荐项目选题      |
| 不知道该学什么     | 学习路线和市场需求脱节                 | 根据近期岗位要求动态生成学习建议  |
| 信息分散        | 招聘平台、面经、薪资、学习路线分散在多个平台      | 统一汇总成方向分析页和周报     |
| 对市场趋势没有感知   | 不知道哪些技能在升温或降温               | 提供周趋势、月趋势和技能变化榜   |

## 7. 产品核心定位

产品不是招聘平台，而是：

> 面向计算机学生的职业方向决策工具。

产品核心价值链：

```text
公开岗位数据
  ↓
岗位标准化
  ↓
技术方向分类
  ↓
技能要求提取
  ↓
岗位趋势分析
  ↓
学习路线建议
  ↓
项目与简历建议
```

一句话介绍：

> 基于真实招聘需求，告诉计算机学生当前哪些方向值得准备、企业要求什么技能、应该学什么、做什么项目。

## 10. 数据处理规则

```text
可信度评分
```

---

## 10.3 薪资标准化

需要处理不同薪资格式：

```text
10-15K
15-25K·14薪
200/天
300-500/天
20-30万/年
面议
```

标准化字段：

```text
salary_min_monthly
salary_max_monthly
salary_median_monthly
salary_months
salary_type
```

薪资类型：

* 月薪；
* 年薪；
* 日薪；
* 实习日薪；
* 面议；
* 未知。

规则：

1. 面议不参与薪资中位数统计。
2. 日薪岗位单独统计，不与正式月薪混算。
3. 年包可换算为月均，但需要保留原始字段。
4. 异常薪资需要过滤。

---

## 10.4 应届友好度判断

判断岗位是否适合应届生。

高置信关键词：

```text
应届生、校招、实习、毕业生、经验不限、接受应届、校园招聘
```

低置信关键词：

```text
1年以内、可培养、初级、助理
```

## 15. 合规与风险控制

### 15.1 数据采集边界

原则：

1. 优先使用公开、低风险数据源；
2. 不绕过登录；
3. 不绕过验证码；
4. 不绕过风控；
5. 不抓取聊天、简历、手机号、联系方式等个人敏感信息；
6. 不完整复制商业招聘平台岗位详情；
7. 对外展示聚合分析，不替代原招聘平台流量；
8. 遵守数据源网站的访问规则；
9. 控制采集频率；
10. 保留数据来源说明。

### 15.2 展示边界

前台展示尽量使用聚合结果：

* 岗位数量；
* 技能词频；
* 薪资区间；
* 城市分布；
* 趋势变化；
* 学习建议。

不重点展示：

* HR 姓名；
* 联系方式；
* 原始岗位完整描述；
* 可替代原平台的岗位详情页。

### 15.3 免责声明

数据说明页需要说明：

```text
本产品基于公开招聘信息样本进行统计分析，结果仅供学习规划和职业方向参考，不构成就业、投资或薪资承诺。
由于数据来源、发布时间、岗位重复、薪资口径等因素，统计结果可能存在误差。
```

## 21. 产品核心差异化

与竞品相比，本产品不做泛招聘，不做单纯查薪资，也不做面经社区。

差异化定位：

| 产品类型           | 解决问题        | 本产品差异          |
| -------------- | ----------- | -------------- |
| BOSS / 智联 / 猎聘 | 找岗位         | 不做岗位搜索，做岗位趋势分析 |
| 牛客             | 刷题、面经、校招讨论  | 不做社区，做方向决策     |
| OfferShow      | 查薪资、看 offer | 不只查薪资，分析技能和方向  |
| 职友集 / 看准网      | 查工资、公司评价    | 聚焦计算机学生学习和就业规划 |
| 培训机构路线图        | 静态学习路线      | 基于近期岗位数据动态更新   |

产品一句话差异化：

> 招聘平台告诉你有哪些岗位，本产品告诉你这些岗位背后意味着什么，以及你该怎么准备。

## 25. 总结

本产品的核心不是抓取更多岗位，而是把岗位数据转化为学生能理解、能执行的职业准备建议。

V1.0 的目标是跑通以下闭环：

```text
公开岗位数据
  ↓
采集与清洗
  ↓
方向和技能分析
  ↓
趋势报告
  ↓
学生访问和传播
  ↓
验证付费意愿
```

只要能持续输出高质量方向分析和学习建议，本产品就有机会在“计算机学生职业规划”这个垂直场景中建立壁垒。
